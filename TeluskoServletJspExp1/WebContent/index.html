You can Either Override a Service method or doGet,doPost Method. The reason for writing in doPost instead of service is because on clients end we can change the method(get/post) from Inspect(ctrl+shft+i) in browser and get our required method say get where data is exposed in url. so we can limit such things using doGet, doPost methods where we are allowing only the corresponding methods. We can use Dispatcher Servlet if both servlet are in same website. If you are dealing with one servlet on our website and the other on diffrent website like payment gateway then go fpr sendRedirect. Remember that in dispatcherServlet we use same req,res. But thats not the case in sendRedirect.
